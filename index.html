// the problem

Online PDF tools ask you
to trust a stranger with your files

Contracts, invoices, medical records, legal documents — these aren't files you should be uploading to a random server. PDF Stitcher runs entirely on your machine using Python and Flask. The processing happens locally, and nothing is ever transmitted.

// built for

Real workflows,
real documents

01

Financial reporting

Combine quarterly statements, invoices, and audit docs without exposing them to third parties.

02

Legal & administrative

Stitch contracts, annexes, and signature pages for clean submission packets.

03

Medical records

Merge prescriptions, lab results, and reports safely — no HIPAA anxiety.

04

Academic & alumni work

Assemble applications, transcripts, and supporting documents in the right order.

// features

Simple by design

Drag & drop

Drop files straight into the browser — no file picker required.

Add in batches

Start with three files, add more later. No need to select everything upfront.

Reorder freely

Move files up or down until the sequence is exactly right.

Remove files

Changed your mind? Remove any file before hitting merge.

Custom filename

Name the output before downloading — not "merged_1732894.pdf".

One-click reset

Clear the queue instantly and start fresh without reloading the page.

// get started

Running in
under a minute

1

Clone the repo

git clone https://github.com/swapnilbawane/pdf-offline

2

Install dependencies

pip install flask pypdf — two packages, nothing else.

3

Start the local server

Run python app.py — the browser opens at localhost:5000 automatically.

4

Merge your PDFs

Add files, reorder if needed, name the output, download. That's the whole workflow.
